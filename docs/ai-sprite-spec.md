# AI Sprite Generation Spec — Dune Experiments

Инструкция для генерации спрайтов юнитов/зданий через AI. Цель: спрайты **чётко
считываются на экране, видны детали**, стиль — пиксель-арт уровня StarCraft 1 /
Dune 2000. Каждое направление готовится **нативно** (без зеркала и без поворота
одного кадра), чтобы сохранить пропорции, фиксированный свет и тень.

> **Главное ограничение, которое надо учитывать.** Сегодняшние AI-генераторы плохо
> держат **консистентность между 32 кадрами** одного юнита (плывут пропорции, цвета,
> детали, центр вращения). Поэтому ниже — не «попроси 32 картинки», а пайплайн с
> опорным кадром + image-to-image + (для сложных случаев) 3D-блокаут как поза-референс.
> Закладывай ручную чистку каждого кадра.

---

## 1. Технический контракт (для ВСЕХ спрайтов)

| Параметр | Значение |
|---|---|
| Стиль | pixel art, жёсткие края, **без сглаживания и градиентов** (только дизеринг), ограниченная палитра |
| Ракурс | высокий 3/4 top-down, **угол камеры ~30° над землёй (pitch), одинаковый для всех ассетов** |
| Свет | один key-light сверху-слева (азимут ~315°, фиксирован), мягкий ambient-fill. **Свет НЕ меняется между кадрами направлений** |
| Фон | **полностью прозрачный** (alpha). Контактную тень в спрайт НЕ запекать — движок рисует её отдельным фикс-эллипсом |
| Холст | квадратный, фиксированный на класс, **юнит центрирован** (центр масс = центр холста) для совпадения пивота |
| Контур | тёмный, почти чёрный (1px в целевом разрешении) по силуэту — отрывает фигуру от песка |
| Контраст | проходит **силуэт-тест** на бледном песке (#c8a85a). Тело — средне-тёмное; самые яркие блики только на верхних-левых гранях |
| Team-color | зоны командного цвета красить **отдельным слоем** или ключевым цветом (чистая magenta), чтобы движок перекрашивал под дом |

### Направления (для подвижных юнитов)
- **32 кадра**, каждый нарисован нативно под своё направление. **Зеркало и поворот запрещены.**
- Пропорции, размер, цвета, плотность деталей — **идентичны** во всех 32 кадрах; меняется только ориентация.
- Конвенция направления (совпадает с движком, `frameForAngle`):
  - кадр **00 = смотрит вправо (восток)**;
  - каждый следующий +11.25° **по часовой** на экране;
  - **08 = вниз (юг)**, **16 = влево (запад)**, **24 = вверх (север)**.

### Размеры (целевые, на экране; рисовать в 3–4× и даунсэмплить nearest)
| Ассет | Футпринт/холст | Примечание |
|---|---|---|
| Construction Yard | 48×48 (2×2 тайла), статичный, 1 кадр | автор 128–192px → даунсэмпл |
| Infantry | ~28px фигура, холст 48×48 | можно слегка выходить за тайл |
| Trike | ~36px, холст 48×48 | |
| Tank — корпус | ~44px, холст 56×56 | |
| Tank — башня | отдельный ассет, холст 56×56, 32 кадра | пивот в центре, ствол вправо на кадре 00 |

## 2. Бриф по юнитам (стартовый набор для стенда)

**Construction Yard** (здание, статичное). Тяжёлая укреплённая постройка: плоская
крыша с угловыми бастионами, центральный кран-мачта, въездные ворота на южной стене,
фракционный баннер. Массивный, «весомый». Team-color: баннер + акценты бастионов.

**Infantry.** Один пехотинец сверху-сбоку: округлый шлем (блик сверху-слева), плечи,
торс, винтовка вперёд (на кадре 00 — вправо). Силуэт-сигнатура: маленький, «человечек
со стволом». Team-color: перевязь/нашивка на торсе.

**Trike.** Лёгкий 3-колёсный багги: открытая рама, водитель, передний пулемёт. Силуэт:
три толстых колеса + узкая рама. Быстрый/лёгкий читаемый профиль. Team-color: полоса
на раме.

**Tank** (корпус + ОТДЕЛЬНАЯ башня). Корпус: гусеницы по бокам, наклонная лобовая
плита, гнездо башни. Башня: куб с люком + длинный ствол (на кадре 00 — вправо).
Силуэт: явно крупнее и «квадратнее» трайка. Team-color: бортовой значок + борт башни.

> Размерная иерархия обязана читаться: **infantry < trike < tank**.

## 3. Пайплайн генерации (как реально получить 32 консистентных кадра)

**Шаг 1 — опорный кадр.** Сгенерировать ОДИН «геройский» кадр в 3/4 (например, юнит
смотрит на юго-восток). Итерировать промпт, пока стиль/контраст/детали не устроят.
Это эталон личности юнита.

**Шаг 2 — остальные направления через image-to-image.** Для каждого из 32 углов:
подать опорный кадр как reference (character/style reference или ControlNet) +
инструкцию «тот же юнит, развёрнут лицом на <угол>, свет сверху-слева неизменен, те же
пропорции и цвета». Это резко повышает консистентность против генерации с нуля.

**Шаг 3 — для сложных/органичных (пехота) или если плывёт.** Сделать грубый 3D-блокаут
(примитивы) в Blender, отрендерить 32 ракурса как **depth/pose-референс** под ControlNet
— AI рисует пиксель-арт «поверх» правильной позы. Это самый надёжный способ удержать
геометрию на всех углах (по сути — гибрид метода SC1).

**Шаг 4 — пост-обработка каждого кадра.**
1. Квантизация палитры (≤ N цветов, общая палитра на юнит).
2. Прозрачный фон, **убрать любую тень с земли** (она в движке).
3. Выровнять пивот (центр юнита = центр холста) — критично для совпадения кадров.
4. Даунсэмпл до целевого размера фильтром **nearest** (без сглаживания).
5. Тёмный контур по силуэту, если AI его размыл.

## 4. Промпт-шаблон (English — для генератора)

**Positive:**
```
pixel art sprite, top-down 3/4 view (~30° camera pitch), single <UNIT> for a
desert RTS in the style of StarCraft 1 / Dune 2000, hard-edged limited palette,
key light from top-left (fixed), facing <DIRECTION>, centered, transparent
background, no ground shadow, crisp readable silhouette, dark near-black outline,
<FACTION> house colors on <TEAM-COLOR ZONES>, high detail density, consistent
proportions
```
**Negative:**
```
anti-aliasing, blur, gradient background, drop shadow on ground, multiple units,
text, watermark, inconsistent proportions, soft focus, 3d render, photorealistic
```
Подставлять: `<UNIT>` (см. бриф), `<DIRECTION>` (e.g. "facing south-east / down /
left"), `<FACTION>` (atreides blue / harkonnen red / ordos green), `<TEAM-COLOR ZONES>`.

## 5. Вывод и интеграция в проект

- **Именование:** `unit_<faction>_<kind>_<NN>.png` (NN = 00..31), башня —
  `unit_<faction>_<kind>_turret_<NN>.png`, статичное здание — `build_<faction>_<kind>.png`.
- Сложить в `assets/sprites/`, упаковать в атлас, грузить через PixiJS `Assets`/`Spritesheet`.
- Движок выбирает кадр по `frameForAngle(frames, u.dir)` (уже есть в `src/render/spritesV2.ts`),
  `N_DIR = 32`.
- **Проверка:** кидать готовые кадры в колонку «СТАЛО» стенда `/compare-preview.html`
  и сравнивать с «БЫЛО» до коммита (силуэт-тест, контраст к песку, плавность поворота,
  тень на месте при вращении).

## 6. Чек-лист приёмки кадра
- [ ] прозрачный фон, нет тени на земле
- [ ] свет сверху-слева (одинаков на всех 32 кадрах)
- [ ] пивот в центре, юнит не «прыгает» между кадрами
- [ ] силуэт читается на песке без цвета
- [ ] team-color зоны на отдельном слое/ключевом цвете
- [ ] пропорции и палитра совпадают со всеми кадрами юнита
- [ ] размерная иерархия infantry < trike < tank соблюдена
